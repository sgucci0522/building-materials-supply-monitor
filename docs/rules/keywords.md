# キーワード設計ルール

## 3層構造

monitor.py には3種類のキーワードリストがある。
優先度順に処理される。

### 1. RESUMPTION_KEYWORDS (供給回復🟢)

最重要。**ぐっちさんが本来知りたいシグナル**。

```python
RESUMPTION_KEYWORDS = [
    "再開", "正常化", "解除", "復旧", "通常", "回復",
    "受注再開", "出荷再開", "供給再開", "納期正常",
]
```

- メーカー側のお知らせで「受注再開」「正常化」「解除」が出れば
  ほぼ間違いなく供給回復シグナル
- このカテゴリでヒットしたら必ず通知

### 2. RESTRICTION_KEYWORDS (供給制限🔴)

参考情報。**状況悪化の早期把握**用。

```python
RESTRICTION_KEYWORDS = [
    "受注停止", "受注制限", "出荷停止", "出荷制限",
    "供給制限", "供給停止", "納期遅延", "新規受注",
]
```

### 3. メーカー固有 keywords (🟡)

`targets.yaml` の `keywords` に書く。
メーカー固有の製品名・ブランド名を入れるのが効果的。

例:
- クボタケミックス: `["塩ビ", "HIVP", "TS継手", "受注"]`
- 三菱ガス化学ネクスト: `["ユピカ", "4190", "ポリエステル"]`
- 田島ルーフィング: `["オルタック", "ルーフィング"]`

## 注意点

- キーワードは多すぎると誤検知の原因
- 「値上げ」だけでは現在の状況下では毎日発火するので注意
- 「販売」「製品」のような一般語は使わない
